1. PROJECT OVERVIEW
================================================================================

"Amy's 80s Movie Vault" is a static web app showcasing a curated collection of
296 80s movies. The data comes from a hand-curated spreadsheet (Excel) that was
converted to JSON. The site is built with plain HTML/CSS/JS (no frameworks, no
build step) and is designed to be hosted for free on GitHub Pages.

Owner: MizzBuffalo (GitHub username)
Repo: https://github.com/MizzBuffalo/Eighties-Movies

DATA SUMMARY (from amys_80s_vault.json):
  - 296 movies total
  - Each movie has: title, year, genres, special_sections, collections, notes
  - 17 genres (Action, Comedy, Drama, Horror, Sci-Fi, etc.)
  - 6 special sections: Cult Classics, Holiday Films, Horror Vault,
    John Hughes Universe, Queer Cinema & Stories, Spielberg Universe
  - 7 collections: Badass Women of the 80s, Brat Pack, Cult Classics,
    Iconic Soundtracks, Sequels & Franchises, Spielberg Universe, Underrated Gems
  - 14 movies flagged as queer_flagged
  - 238 of 296 movies have handwritten "notes" (Amy's personal commentary)

4. ARCHITECTURE & STRATEGY DECISIONS
================================================================================

PIVOT FROM REACT TO PLAIN HTML/JS:
  The original PLAN.md proposed React + Vite + Tailwind + TypeScript. This was
  abandoned in favor of plain HTML/CSS/JS because:
  - Amy (the site owner) is not a developer
  - She needs to be able to open files in a browser with no build step
  - 296 movies is small enough that no framework is needed
  - Single-file HTML apps are easy to share and host
  - No npm, no Node.js, no build pipeline required for the end user
  ** PLAN.md still contains the old React plan. SETUP_ADVICE.md has the actual
     approach being followed. **

SINGLE-FILE ARCHITECTURE:
  Each vault HTML file is completely self-contained:
  - All CSS is inline in <style> tags
  - All JS is inline in <script> tags
  - All 296 movies are embedded as a MOVIES[] array in the HTML
  - Only external dependencies: Google Fonts CDN + TMDB image CDN
  - No separate .css or .js files needed at runtime

TMDB POSTER STRATEGY (TWO APPROACHES EXIST):
  Approach 1 — Pre-fetch with Node.js (recommended):
    Run fetch_all_posters.js with a TMDB API key to enrich all vault files
    with poster_path, overview, vote_average, director, and cast data.
    Movies then load posters from TMDB's public image CDN (no API key needed
    at runtime). Cache stored in poster_cache.json.

  Approach 2 — Client-side enrichment (current fallback):
    vault.html includes a client-side enrichment function that:
    - On page load, checks each movie for missing poster/cast/director data
    - Calls TMDB API directly from the browser using a hardcoded API key
    - Caches results in localStorage
    - Updates the DOM as data arrives
    This works but exposes the API key in the HTML source.

  The TMDB API key currently hardcoded in the files:
    Bearer token in vault.html (client-side)
    API key in fetch_all_posters.js: b5be0091e28a74a864c82ec31da4f981

7. TECHNICAL DETAILS FOR CONTINUING DEVELOPMENT
================================================================================

VAULT.HTML INTERNAL ARCHITECTURE:
  The file is ~1293 lines, structured as:

  Lines 1-7:      HTML head, meta tags, Google Fonts link
  Lines 8-350:    <style> block — all CSS (Neon Arcade theme, responsive grid,
                  modal, filters, dashboard, scanlines, etc.)
  Lines 351-750:  <script> — MOVIES[] array (all 296 movies as JS objects)
                  Each movie: { title, year, genres[], special_sections[],
                  collections[], notes, queer_flagged, poster_path, overview,
                  vote_average, director, cast[] }
  Lines 750-850:  HTML body — header, dashboard container, filter bar, grid
                  container, modal template
  Lines 850-1293: <script> — App logic:
                  - buildDashboard() — renders collection cards
                  - buildFilters() — renders genre/year/section/collection pills
                  - buildGrid() — renders all 296 movie cards
                  - applyFilters() — combines all active filters
                  - openModal(idx) — shows movie detail overlay
                  - enrichAllMovies() — client-side TMDB data fetching
                  - enrichMovie(movie, idx) — single movie TMDB lookup
                  - localStorage caching for TMDB data

TMDB API USAGE:
  Search endpoint: GET https://api.themoviedb.org/3/search/movie
    ?query={title}&year={year}
    Headers: Authorization: Bearer {token}
  Credits endpoint: GET https://api.themoviedb.org/3/movie/{tmdb_id}/credits
    Headers: Authorization: Bearer {token}
  Image CDN (no auth needed): https://image.tmdb.org/t/p/w300/{poster_path}
    Sizes available: w92, w154, w185, w300, w342, w500, w780, original

FILTER SYSTEM:
  All filtering happens client-side on the MOVIES[] array.
  Active filters stored as JS variables, combined with AND logic.
  DOM elements are shown/hidden via CSS classes (not removed/recreated).
  Results count updates in real-time: "Showing X of 296 films"

DEPENDENCIES (runtime):
  - Google Fonts CDN (Press Start 2P, Space Mono)
  - TMDB image CDN (image.tmdb.org) for poster images
  - No npm packages, no frameworks, no build tools needed

DEPENDENCIES (development/build scripts only):
  - Node.js (for running fetch_all_posters.js)
  - TMDB API key (free, for poster pre-fetching)
